<objective>
Build the parametric `<KeyboardGrid>` component (D14.3) and refactor the existing per-key surfaces (NoteGrid, computeNoteMap) to render `TOTAL_KEYS` cells based on the active variant. After this plan, switching between T16 and T32 in `useVariant()` immediately re-renders all per-key array editors with the correct row count.
</objective>

<must_haves>
  <truths>
    - `<KeyboardGrid>` is the single grid container — NoteGrid (and any future per-key editor in v202+) composes it. Cell content is supplied by the caller via `renderKey={(index) => ReactNode}`.
    - Column count is invariant: `cols = 4` for both variants (UI-SPEC §Component Contract). T32's 4×8 means 8 ROWS, not 8 columns. Never reflow to 8×4 on small viewports — the physical-mapping invariant is non-negotiable.
    - The grid `gap`, cell tap target (44px min), and T16 `max-width: 280px` come from existing NoteGrid CSS — copy them verbatim, do not invent new values.
    - T32 max-width is `100%` (UI-SPEC §Spacing exceptions); on viewport `< 480px` the parent card applies `overflow-x: auto` and the grid keeps `min-width: 280px`.
    - `computeNoteMap()` is the only function with a hardcoded loop bound of 16 — every other per-key path uses the array length of `noteMap`. Parametrizing `computeNoteMap(...args, totalKeys)` propagates correctness through the rest of the chain.
    - The `aria-label` MUST be `"Keyboard layout: {variant}, {keys} keys"` (UI-SPEC copywriting contract).
    - No new design tokens. Reuse `--space-sm`, `--gray-200`, etc.
    - This plan does NOT add a calibration view — calibration page does not yet exist as a per-key editor in editor-tx; it stays out-of-scope until that page lands. Spec D14.3 says "Calibration view branches on `variant` for the count only — same component for both"; when the calibration view is added, it will reuse `<KeyboardGrid>` automatically.
    - Existing Dashboard `max={16}` (line 147) is for custom-scale-degree count, NOT per-key — leave unchanged (Research §4).
  </truths>
  <nonneg>
    - `npm run typecheck`, `npm run lint`, `npm run test` all clean after this plan.
    - No regression in existing NoteGrid tests on T16: 16 cells render with the same MIDI note labels and degree colors as before.
  </nonneg>
</must_haves>

    1. `editor-tx/src/components/KeyboardGrid/KeyboardGrid.tsx` with EXACTLY this API and structure:
    ```tsx
    import type { ReactNode } from 'react'
    import styles from './KeyboardGrid.module.css'

    export type KeyboardGridProps = {
      keys: number
      cols: number
      rows: number
      renderKey: (index: number) => ReactNode
      'aria-label': string
    }

    export function KeyboardGrid({ keys, cols, rows, renderKey, ...rest }: KeyboardGridProps) {
      // Defensive: keys must equal cols * rows. Throw in dev for early catch.
      if (cols * rows !== keys && import.meta.env.DEV) {
        // eslint-disable-next-line no-console
        console.error(`[KeyboardGrid] cols (${cols}) * rows (${rows}) !== keys (${keys})`)
      }
      const cells: ReactNode[] = []
      for (let i = 0; i < keys; i++) {
        cells.push(<div key={i} className={styles.cell}>{renderKey(i)}</div>)
      }
      return (
        <div
          role="grid"
          aria-label={rest['aria-label']}
          className={styles.grid}
          data-keys={keys}
          data-cols={cols}
          style={{ gridTemplateColumns: `repeat(${cols}, 1fr)` }}
        >
          {cells}
        </div>
      )
    }
    ```
