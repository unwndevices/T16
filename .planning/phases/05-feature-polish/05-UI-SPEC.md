## Color

Inherited from Phase 3. Additions for this phase noted below.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--gray-50` (#faf9fe) | Page background |
| Secondary (30%) | `--gray-800` (#202022) nav, `--white` (#ffffff) cards | Navigation bar, card surfaces |
| Accent (10%) | `--primary-800` (#6246ea) | Primary CTA buttons, active nav indicator, sync status dot (synced), focused input borders |
| Destructive | `--color-error` (#e4412c) | Error toasts, destructive actions |

Accent reserved for: primary CTA buttons, active navigation indicator, focused input rings, synced-state status dot, MIDI monitor pause/resume toggle active state.

### Note Grid Scale Degree Colors

HSL hue rotation for scale degrees. Root degree anchored at 270 (purple, aligned with `--primary-800`). Remaining degrees distributed evenly across 360 degrees.

| Degree | Hue | HSL | Usage |
|--------|-----|-----|-------|
| Root (1) | 270 | hsl(270, 70%, 55%) | Root note cells |
| 2nd | 210 | hsl(210, 70%, 55%) | Second degree cells |
| 3rd | 150 | hsl(150, 70%, 55%) | Third degree cells |
| 4th | 90 | hsl(90, 70%, 55%) | Fourth degree cells |
| 5th | 30 | hsl(30, 70%, 55%) | Fifth degree cells |
| 6th | 330 | hsl(330, 70%, 55%) | Sixth degree cells |
| 7th | 0 | hsl(0, 70%, 55%) | Seventh degree cells |

For scales with more than 7 degrees (e.g. Chromatic): cycle hues. Formula: `hsl(((index * 360 / degreeCount) + 270) % 360, 70%, 55%)`.

Out-of-scale keys: `--gray-200` background, `--gray-400` text. No hue.

Cell background: the degree HSL color at 15% opacity. Cell border-left: 3px solid at full opacity. Text: `--gray-800`.

### MIDI Monitor Message Colors

| Message Type | Indicator Color | Usage |
|-------------|----------------|-------|
| Note On | `--primary-800` (#6246ea) | Left border accent on note-on rows |
| Note Off | `--gray-400` (#adacb6) | Left border accent on note-off rows |
| CC | `--color-success` (#2096a6) | Left border accent on CC rows, CC bar fill |

CC bar background: `--gray-200`. CC bar fill: `--color-success` at width proportional to value/127.
Note velocity bar background: `--gray-200`. Note velocity bar fill: `--primary-800` at width proportional to velocity/127.

## Component Inventory (Phase 5 Additions)

### NoteGrid

4x4 CSS Grid component placed in the Dashboard Keyboard tab.

| Property | Value |
|----------|-------|
| Grid | `grid-template-columns: repeat(4, 1fr)` |
| Gap | `--space-sm` (8px) |
| Cell min size | 44px x 44px |
| Cell border-radius | `--radius-md` (8px) |
| Cell padding | `--space-sm` (8px) |
| Max width | 280px (4 cells + 3 gaps = 4*62 + 3*8) |

Cell layout (vertical stack, centered):
1. Note name (e.g. "C4") -- Label size, semibold
2. MIDI number (e.g. "60") -- 10px, regular, `--gray-400`

States:
- In-scale: degree-colored background (15% opacity), 3px left border at full degree color
- Out-of-scale: `--gray-200` background, no left border, `--gray-400` text
- Active (MIDI note playing): scale to 0.95 + full opacity background, transition 100ms

### MidiMonitor

New page component at `/monitor` route.

| Property | Value |
|----------|-------|
| Layout | Single column, max-width 960px centered |
| Message list | Scrollable container, max-height `calc(100vh - 200px)` |
| Message row height | 40px minimum |
| Message row padding | `--space-sm` (8px) horizontal, `--space-xs` (4px) vertical |
| Message row border-left | 3px solid (color by message type) |
| Message row background | `--white`, alternate rows `--gray-50` |
| Bar visualization width | 120px fixed |
| Bar height | 16px |
| Bar border-radius | `--radius-sm` (4px) |

Toolbar (sticky top, below nav):
- "Clear" button: secondary variant, icon `MdDelete`
- "Pause" / "Resume" toggle: secondary variant, icons `MdPause` / `MdPlayArrow`
- Message count badge: Label size, `--gray-500`

Message row layout (horizontal, centered vertically):
1. Timestamp (10px mono, `--gray-400`, 60px width)
2. Type badge (Label size, 60px width, colored by type)
3. Message detail (Body size mono, flex 1)
4. Value bar (120px, right-aligned)

Max entries: 100. New messages prepended at top. Auto-scroll when not paused.

### PWA Components

No new visible UI components. PWA is infrastructure only:
- `vite-plugin-pwa` configuration in `vite.config.ts`
- Web App Manifest (`manifest.webmanifest`)
- Offline indicator: when `navigator.onLine === false`, show a banner below NavBar

Offline banner:
- Background: `--color-warning` (#eaf51a) at 20% opacity
- Text: "You're offline. Device communication requires a connection." -- Body size, `--gray-700`
- Height: 36px, centered text
- Dismissible: no (auto-hides when back online)

### Interaction States (new components)

All new components inherit the Phase 3 interaction state contract:

| State | Visual Treatment |
|-------|-----------------|
| Default | `--gray-200` border, `--gray-50` background |
| Hover | `--gray-300` border, `--gray-100` background |
| Focus | 2px `--primary-800` ring, 2px offset |
| Active/Pressed | `--primary-900` background on buttons |
| Disabled | 50% opacity, cursor not-allowed |

Transitions: `150ms ease-out` for all state changes. Note grid cell active pulse: `100ms ease-out`.

## Accessibility Contract

### Note Grid
- Each cell has `role="gridcell"` within a `role="grid"` container
- `aria-label` on each cell: "{note name} - {in scale | out of scale}" (e.g. "C4 - root note")
- Grid is keyboard-navigable with arrow keys
- Color is never the sole indicator: out-of-scale cells also have muted text weight

### MIDI Monitor
- Message list uses `role="log"` with `aria-live="polite"` (when not paused)
- Pause button toggles `aria-live` between "polite" and "off"
- Value bars have `role="meter"` with `aria-valuenow`, `aria-valuemin="0"`, `aria-valuemax="127"`
- Clear button: no confirmation needed (non-destructive, messages are transient)

### Offline Banner
- `role="status"` with `aria-live="polite"`
