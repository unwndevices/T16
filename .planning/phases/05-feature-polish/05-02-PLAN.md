    **Create `editor-tx/src/hooks/useMidiMonitor.ts`:**

    Export interface `MidiMessage`:
    ```typescript
    export interface MidiMessage {
      id: number
      timestamp: number
      type: 'cc' | 'noteon' | 'noteoff'
      channel: number
      data1: number  // note number or CC number
      data2: number  // velocity or CC value
    }
    ```

    Export `const MAX_MESSAGES = 100`

    Export function `useMidiMonitor(input: Input | null)`:
    - State: `messages: MidiMessage[]`, `paused: boolean`
    - Ref: `idRef = useRef(0)` for auto-incrementing message IDs
    - useEffect depending on [input, paused]:
      - If !input or paused, return early (cleanup only)
      - Create handler for 'controlchange': extract channel from `e.message.channel`, controller number from `e.controller.number`, value from `e.value` (raw 0-127 integer -- use `e.rawValue ?? Math.round(e.value * 127)` since webmidi v3 normalizes to 0-1 for .value). Actually check webmidi v3 API: `.controller.number` for CC#, `.rawValue` for 0-127 raw value, `.message.channel` for channel.
      - Create handler for 'noteon': channel from `e.message.channel`, note from `e.note.number`, rawAttack from `e.rawAttack` (0-127)
      - Create handler for 'noteoff': channel from `e.message.channel`, note from `e.note.number`, data2=0
      - Each handler: increment idRef, create MidiMessage with Date.now() timestamp, prepend to messages, slice to MAX_MESSAGES
      - Register listeners, return cleanup that removes them
    - `clear` function: setMessages([])
    - Return { messages, paused, setPaused, clear }

    Test using a mock Input object. Since webmidi Input uses addListener/removeListener pattern, create a mock that stores listeners and allows manual triggering.
    - Test: adding a CC message produces correct MidiMessage
    - Test: adding a noteon message produces correct MidiMessage
    - Test: messages are capped at MAX_MESSAGES (add 101, verify length is 100, oldest dropped)
    - Test: clear() empties messages
    - Test: when paused, no messages added (verify by triggering events after pause)
