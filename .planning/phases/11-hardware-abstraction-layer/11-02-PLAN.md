# Phase 11 — Plan 02: LED Layer Migration

## Goal

Replace `kMatrixWidth`, `kMatrixHeight`, `sliderLength`, `NUM_LEDS` macros (and inline literal `16` for `patternleds` size) with `variant::CurrentVariant::kConfig.*` constexpr access. T16 build behavior must be identical (4-wide, 4-tall matrix, 24 LEDs total).

## Context

- RESEARCH §1.2: enumerated LED-layer macros and call sites.
- RESEARCH §5: type compatibility of constexpr replacements (uint8_t, allowed in array bounds).
- CONTEXT D11.2: legacy macros deleted outright in this phase — no alias-then-remove.

## Tasks
