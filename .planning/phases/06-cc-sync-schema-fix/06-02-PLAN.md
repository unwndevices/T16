<objective>
Fix CC per-parameter sync by adding a dedicated 5-byte SysEx send path for CC domain updates, replacing the broken 4-byte generic path.

Purpose: The web editor sends 4 bytes for CC param updates but firmware expects 5 (domain, bank, ccIndex, channel, id). The firmware silently rejects all CC edits. This plan adds a dedicated CC send function, a new reducer action, and updates the Dashboard handlers to always send channel+id together atomically.

Output: CC edits in the Dashboard reach the firmware and update device state.
</objective>
