# Plan 12.03 — Dual-mux ADC scan + per-channel keyMapping translation

## Objective

Implement the actual dual-mux scan in `Adc::InitMuxes` and `Adc::ReadValues` so the T32 reads both muxes per channel and the `keyMapping[ch]` lookup translates electrical channels to logical key indices. Implements **D12.1**, addresses requirements **T32-01**, **T32-02**, and operationalizes **T32-03** (the data lives in 12.02; this plan consumes it). T16 single-mux behavior MUST remain identical.

## Background

Current state (`src/Libs/Adc.cpp`):
- `InitMuxes` stores only mux 0 — second mux is ignored.
- `ReadValues` only reads `_config._pin` — single mux.
- `_channels[iterator]` is written directly without translation through `keyMapping`.

3dot0 reference (`origin/3dot0:src/Libs/Adc.cpp`):
- `_pin` and `_pin2` for two common pins.
- `ReadValues`: `SetMuxChannel(iterator); analogRead(_pin) → _channels[iterator]; if (_pin2) analogRead(_pin2) → _channels[iterator + 16]; iterator++; if (iterator >= _channels.size()/2) iterator = 0;`
- No `keyMapping` translation (3dot0 stores `mux_idx` per Key instead).

This plan diverges from 3dot0 in one important way: instead of a bare `_pin`/`_pin2` pair, store all `MultiplexerConfig` entries in a fixed-size `std::array<MultiplexerConfig, kMaxMuxes>` (or copy the pointer + count) so the scan loop can iterate dynamically. This is the v1.0-architecture rewrite mandated by D12.1.

<task id="12.03.3" type="execute">
  <action>
    Rewrite `Adc::InitMuxes` in `src/Libs/Adc.cpp` to store ALL muxes (not just mux 0) and apply `useSharedSelect`. Replace the entire current `InitMuxes` body (lines 50–88) with:

    ```cpp
    void Adc::InitMuxes(const MultiplexerConfig *muxes, uint8_t mux_count)
    {
        if (mux_count == 0) return;
        if (mux_count > kMaxMuxes) mux_count = kMaxMuxes;
        _mux_count = mux_count;

        // Store mux configs for ReadValues() to iterate.
        for (uint8_t m = 0; m < mux_count; ++m) {
            _muxes[m] = muxes[m];
        }

        // Configure select pins ONCE — first mux drives them. Any subsequent mux
        // with useSharedSelect=true reuses the same lines.
        analogSetAttenuation(ADC_6db);
        for (uint8_t i = 0; i < 4; ++i) {
            uint8_t pin = static_cast<uint8_t>(muxes[0].selectPins[i]);
            _mux_pin[i] = pin;
            if (pin != 0) pinMode(pin, OUTPUT);
        }

        // Configure each mux's commonPin (and any non-shared select pins for muxes
        // that explicitly do NOT share — useSharedSelect=false on a non-first mux).
        for (uint8_t m = 0; m < mux_count; ++m) {
            uint8_t com = static_cast<uint8_t>(muxes[m].commonPin);
            if (com != 0 && static_cast<int8_t>(muxes[m].commonPin) != -1) {
                pinMode(com, INPUT);
            }
            if (m > 0 && !muxes[m].useSharedSelect) {
                // Non-shared select pins for a secondary mux: configure them.
                for (uint8_t i = 0; i < 4; ++i) {
                    uint8_t pin = static_cast<uint8_t>(muxes[m].selectPins[i]);
                    if (pin != 0) pinMode(pin, OUTPUT);
                }
            }
            // m == 0 OR useSharedSelect=true → skip (already done above OR inherited).
        }

        // Mirror first mux pin into legacy _config._pin for Init()-era callers.
        _config._pin = static_cast<uint8_t>(muxes[0].commonPin);
        for (uint8_t i = 0; i < 4; ++i) _config._mux_pin[i] = _mux_pin[i];

        // Total channels = sum of keyMapping sizes (= TOTAL_KEYS for the variant).
        _total_channels = 0;
        for (uint8_t m = 0; m < mux_count; ++m) {
            _total_channels += static_cast<uint8_t>(muxes[m].keyMapping.size());
        }
        _channels.clear();
        for (uint8_t i = 0; i < _total_channels; ++i) {
            _channels.push_back(AdcChannel());
        }
    }
    ```


<task id="12.03.4" type="execute">
  <action>
    Rewrite `Adc::ReadValues()` in `src/Libs/Adc.cpp` to scan all muxes per channel and apply `keyMapping[ch]` translation.

    Replace the entire current `ReadValues()` (lines 250–265) with:

    ```cpp
    void Adc::ReadValues()
    {
        // Set select pins once for this channel (shared across all muxes that
        // declare useSharedSelect=true).
        SetMuxChannel(iterator);

        // Read every mux's commonPin BEFORE incrementing the channel iterator.
        for (uint8_t m = 0; m < _mux_count; ++m) {
            uint8_t logical_key = _muxes[m].keyMapping[iterator];
            uint8_t com_pin = static_cast<uint8_t>(_muxes[m].commonPin);
            uint16_t rawValue = analogRead(com_pin);
            uint16_t filteredValue = ApplyFilter(rawValue, logical_key);

            _channels[logical_key].raw = rawValue;
            _channels[logical_key].filtered = filteredValue;
            _channels[logical_key].value = max(
                min(map(filteredValue,
                        _channels[logical_key].minVal,
                        _channels[logical_key].maxVal,
                        0, 4095) / 4095.0f, 1.0f), 0.0f);
        }

        iterator++;
        if (iterator >= 16) iterator = 0;   // 16 = channels per mux, NOT total
    }
    ```

    Two critical correctness points (both verified by static review BEFORE flashing):

    1. **Iterator wraps at 16, not at `_channels.size()`** — the prior code wrapped at total channels (correct only when total = 16); the dual-mux scan must iterate channels 0..15 and read all muxes per iteration.
    2. **Translation through `keyMapping`** — `_channels[logical_key]` instead of `_channels[iterator]`. T16 (identity mapping `keyMapping[ch] = ch`) is unaffected; T32 routes the sample to its logical key slot.

    Also update `Adc::SetMuxChannel` if it currently rejects channels >= 16 — it should accept the same 0..15 range, so the existing implementation is fine. Confirm by reading the existing code.

    Also: `GetMux(uint8_t chn, uint8_t index)` semantics need a sanity-pass. Currently `return _channels[chn + index].value`. Callers consume `chn=mux_idx*16` historically. With the new translation, `_channels[]` is indexed by logical key, so callers should now use `Get(logical_key)`. Audit call sites:

    ```bash
    grep -rn 'GetMux\|GetMuxRaw' src/
    ```

    Document each call site in this task's verification block. If any call site still passes `mux_idx*16 + ch` semantics, fix it to use `Get(logical_key)`. Phase 11 likely converted Keyboard.hpp / Keyboard.cpp already; if not, they MUST be fixed here.

## Risks

- **T16 regression:** the iterator-wrap change (was `>= _channels.size()`, now `>= 16`) and the `keyMapping[ch]` translation are behavior-preserving for T16 ONLY because `_channels.size() == 16` and `keyMapping` is the identity. Both invariants are checked by `pio run -e esp32s3` plus the static_assert pattern. Hardware T16 smoke test is part of milestone-end batch (Plan 12.04 surfaces this).
- **`COM2 = 17` is a placeholder.** Hardware verification (12.03.5.A) MUST confirm or correct it before declaring T32 complete. Software gates pass regardless.
