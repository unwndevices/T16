# Plan 12.04 — Calibration scale-up + scan-time instrumentation + bring-up smoke

## Objective

Generalize the calibration data structures and routine from a hard-coded 16 keys to `variant::CurrentVariant::kConfig.TOTAL_KEYS`, add the scan-time instrumentation that operationalizes the **T32-01 success gate** (per CONTEXT D12.4), and define the end-of-milestone hardware smoke procedure for both T16 (regression) and T32 (bring-up). Implements **T32-04** and the binding portion of **T32-01**.

## Background

Current state (after Phase 11):
- `src/Configuration.hpp:14-15` — `CalibrationData::minVal[16]`, `maxVal[16]`. Hard-coded 16.
- `src/AppEngine.cpp:82, 86, 89, 108, 113` — pass literal `16` everywhere.
- `src/services/CalibrationService.cpp` — runs calibration loop with `numKeys` parameter (already parameterized) but call sites supply `16`.
- `src/Libs/Adc.cpp` — no scan-time logging.

Phase 11 prepared the constants. This plan finishes the conversion at the calibration call sites and adds the timing logger.

## Risks

- **CalibrationData size change is binary-incompatible with on-device files.** A T16 unit upgraded from a pre-Phase-11 build (where `minVal[16]` was the literal-sized `16`) reads the same on-disk schema (the JSON file stores `numKeys` items; `LoadArray` reads `numKeys` of them). No on-disk migration is required.
- **`Key keys_[]` size change in AppEngine.hpp:** verify all consumers tolerate the larger array on T32. Phase 11 should have done this; if not, this plan flushes out the regression.
