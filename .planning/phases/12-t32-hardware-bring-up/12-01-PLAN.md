# Plan 12.01 — Per-variant calibration filename + legacy migration

## Objective

Replace the hard-coded `/calibration_data.json` filename with a per-variant filename derived from `variant::CurrentVariant::kConfig.NAME`. Rename legacy files on first boot so existing T16 users don't lose calibration. Implements **D12.2** and addresses requirement **T32-04** (persistence).

## Background

CONTEXT D12.2 locks: T16 → `/calibration_t16.json`, T32 → `/calibration_t32.json`. A variant swap (re-flash with the other binary) reads no stale data — the file simply doesn't exist and calibration runs.

Three call sites currently reference `/calibration_data.json`:
- `src/AppEngine.cpp:80` — boot-time load
- `src/services/CalibrationService.cpp:123` — calibration write
- `src/SysExHandler.cpp:220` and `:242` — SysEx-triggered calibration reset (deletes file)

`LittleFS.rename(from, to)` is supported on the active ESP32 Arduino core (see `12-RESEARCH.md` §2).

## Tasks

<task id="12.01.1" type="execute">
  <action>
    Add a single shared header constant for the active calibration filename. Create a new inline constexpr in `src/variant_config.hpp` (or extend `variant.hpp`) — exact placement: append to `src/variant.hpp` after the `CurrentVariant` namespace alias:

    ```cpp
    namespace variant {
      // Per-variant calibration filename. Format: "/calibration_<lowercase_name>.json".
      // Defined as a function (not a string literal) so we can derive it from
      // CurrentVariant::kConfig.NAME at compile time without runtime allocation.
      inline const char* CalibrationFilePath() {
        if constexpr (variant::CurrentVariant::kConfig.NAME[0] == 'T' &&
                      variant::CurrentVariant::kConfig.NAME[1] == '3') {
          return "/calibration_t32.json";
        } else {
          return "/calibration_t16.json";
        }
      }
      inline constexpr const char* kLegacyCalibrationPath = "/calibration_data.json";
    } // namespace variant
    ```

    Replace the three string literals `/calibration_data.json` in the source with `variant::CalibrationFilePath()`:
      - `src/AppEngine.cpp:80` — `DataManager calibration("/calibration_data.json");` → `DataManager calibration(variant::CalibrationFilePath());`
      - `src/services/CalibrationService.cpp:123` — same replacement.
      - `src/SysExHandler.cpp:220, :222, :242, :244` — replace both `LittleFS.exists` and `LittleFS.remove` literals.

    Add `#include "variant.hpp"` to any of these files that doesn't already transitively include it (CalibrationService.cpp likely needs it explicitly).

<task id="12.01.2" type="execute">
  <action>
    Add a one-shot legacy-filename migration helper called once during `AppEngine::init()` BEFORE the existing `DataManager calibration(variant::CalibrationFilePath())` line at `src/AppEngine.cpp:80`.

    Implementation: define an inline free function (or static AppEngine method) `MigrateLegacyCalibrationFile()`:

    ```cpp
    static void MigrateLegacyCalibrationFile() {
      const char* current = variant::CalibrationFilePath();
      // Only migrate for T16 — T32 hardware is brand-new, no legacy file exists.
      if (variant::CurrentVariant::kConfig.NAME[1] == '3') return;
      if (LittleFS.exists(current)) return;                              // already migrated
      if (!LittleFS.exists(variant::kLegacyCalibrationPath)) return;     // nothing to migrate
      bool ok = LittleFS.rename(variant::kLegacyCalibrationPath, current);
      if (ok) {
        Serial.printf("Migrated %s -> %s\n", variant::kLegacyCalibrationPath, current);
      } else {
        Serial.printf("WARN: rename %s -> %s failed\n", variant::kLegacyCalibrationPath, current);
      }
    }
    ```

    Place the function definition either inside `src/AppEngine.cpp` in the anonymous namespace (preferred — keeps it file-private and away from the public header) or in a new `src/services/CalibrationMigration.{hpp,cpp}` pair if pattern-mapper output suggests services live in their own files. Default to the anonymous-namespace approach for minimum churn.

    Insert the call as the FIRST line under the `// Calibration data load` comment in `AppEngine::init()`, immediately before `DataManager calibration(...)`.

    Required header includes in AppEngine.cpp: `#include <LittleFS.h>` (likely already transitively included via DataManager — confirm by grepping).
  </action>
