**SerialCommandManager (D-08):**

1. Create `src/services/SerialCommandManager.hpp` with `#pragma once`, `namespace t16`:
   ```cpp
   class SerialCommandManager {
   public:
       using CommandHandler = void(*)(const char* args);

       void registerCommand(const char* name, const char* help, CommandHandler handler);
       void update();  // Called each loop — reads Serial input, dispatches on newline

   private:
       struct Command {
           const char* name;
           const char* help;
           CommandHandler handler;
       };
       std::array<Command, 16> commands_;
       uint8_t commandCount_ = 0;
       char inputBuffer_[64] = {0};
       uint8_t inputPos_ = 0;

       void dispatch(const char* line);
       void printHelp();
   };
   ```
   Use function pointer instead of `std::function` to avoid heap allocation on ESP32 (per 02-RESEARCH.md).

2. Create `src/services/SerialCommandManager.cpp`:
   - `update()`: read available Serial bytes into inputBuffer_. On '\n' or '\r', null-terminate and call `dispatch(inputBuffer_)`, then reset inputPos_.
   - `dispatch()`: compare first word against registered command names using `strcmp`. If match, call handler with remainder as args. If no match, print "Unknown command. Type 'help' for list."
   - `registerCommand()`: add to commands_ array if space available.
   - `printHelp()`: iterate commands_, print name and help string.
   - Register "help" command in constructor or via a method.
