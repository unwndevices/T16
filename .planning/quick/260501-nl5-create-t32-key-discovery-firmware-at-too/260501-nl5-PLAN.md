<objective>

The firmware lights one LED at a time, waits for the user to press the corresponding key, records which mux channel responded, and at the end prints a ready-to-paste `k3dot0Keys[32]` array.

Purpose: Replace the placeholder `k3dot0Keys[]` in `src/variant_t32.hpp` with empirically-validated values without manual probing. Output is for human copy-paste; this firmware does NOT modify the repo and does NOT touch the main firmware build.

<task type="auto">
  <name>Task 2: Implement discovery firmware (src/main.cpp)</name>
  <files>tools/key_discovery/src/main.cpp</files>
  <action>
Create `tools/key_discovery/src/main.cpp` implementing the full discovery firmware. Structure:

**Includes:**
```cpp
#include <Arduino.h>
#include <FastLED.h>
#include "pinout_t32.h"
```

**Constants:**
```cpp
constexpr uint8_t  kNumKeys        = 32;
constexpr uint8_t  kNumLeds        = 32;          // matrix only
constexpr uint8_t  kNumMuxChannels = 16;
constexpr uint8_t  kNumMuxes       = 2;
constexpr uint16_t kPressDelta     = 800;         // ADC delta over baseline counted as a press
constexpr uint32_t kBaselineMs     = 500;
constexpr uint32_t kPressHoldMs    = 50;          // press must be sustained this long
constexpr uint32_t kReleaseMs      = 30;          // require return-to-baseline this long before advancing
constexpr uint16_t kReleaseDelta   = 200;         // back within this delta of baseline = released
constexpr uint16_t kSelectSettleUs = 8;
constexpr uint32_t kModeHoldBootMs = 50;          // debounce for boot-time MODE check
```

**Globals:**
```cpp
CRGB leds[kNumLeds];
uint16_t baseline[kNumMuxes * kNumMuxChannels] = {0};
uint8_t  discovered[kNumKeys];                    // discovered[i] = mux*16 + ch for key i
```

**Helpers:**
- `void selectChannel(uint8_t c)` — write c's low 4 bits to S0..S3 then `delayMicroseconds(kSelectSettleUs)`.
- `uint16_t readMux(uint8_t mux, uint8_t c)` — selectChannel(c); analogRead(mux==0 ? pinout::t32::COM : pinout::t32::COM2). For convenience, also a `void scanAll(uint16_t out[kNumMuxes*kNumMuxChannels])` that does both muxes in one pass.
- `bool modePressed()` — `digitalRead(pinout::t32::MODE) == LOW` (active-low, INPUT_PULLUP).
- `void setAllLeds(CRGB c)` and `void showOnlyLed(uint8_t i, CRGB c)` (clears others, sets leds[i]=c, FastLED.show()).

**setup():**
1. `Serial.begin(115200);` then `delay(200);` and a banner print.
2. `pinMode(MODE, INPUT_PULLUP);`.
3. If `modePressed()` for `kModeHoldBootMs` continuously → `Serial.println("MODE held at boot — restarting"); delay(50); ESP.restart();`.
4. `pinMode(S0..S3, OUTPUT);` and drive low.
5. `pinMode(COM, INPUT); pinMode(COM2, INPUT);` (analog read does not require explicit mode on ESP32-S3, but be explicit). `analogReadResolution(12);`.
6. `FastLED.addLeds<WS2812, pinout::t32::LED_PIN, GRB>(leds, kNumLeds); FastLED.setBrightness(64); setAllLeds(CRGB::Black); FastLED.show();`.
7. **Baseline scan:** loop for `kBaselineMs` accumulating samples per channel into a sum + count, then divide → `baseline[]`. Print "Baseline captured." to Serial along with min/max for sanity.

**loop() — single-shot discovery state machine:**
Use a static `bool done = false;` and a counter `uint8_t i = 0;`. Don't actually rely on Arduino's loop reentry; just do the whole thing once inside a `while (i < kNumKeys)` block, then enter a steady "done blink" loop. Concretely:

```cpp
void loop() {
  static bool ran = false;
  if (!ran) {
    ran = true;
    runDiscovery();
    emitArray();
  }
  // Done indicator: slow green blink on all LEDs forever.
  static uint32_t last = 0;
  static bool on = false;
  if (millis() - last >= 700) {
    last = millis();
    on = !on;
    setAllLeds(on ? CRGB(0, 60, 0) : CRGB::Black);
    FastLED.show();
  }
}
```

**runDiscovery():**
For `i = 0; i < kNumKeys; ++i`:
1. `showOnlyLed(i, CRGB::Red); FastLED.show();` and print `"Waiting for key " + i`.
2. Inner loop: continuously `scanAll()`, find any channel `c` whose reading exceeds `baseline[c] + kPressDelta`. Track `pressStartMs` per candidate. When ONE channel sustains the threshold for `kPressHoldMs` continuously, lock it in as `discovered[i]`.
   - If during the wait `modePressed()` (with debounce ~30ms), print `"Re-arming key " + i`, reset all per-iteration state, and continue waiting.
   - If multiple channels exceed threshold simultaneously, pick the one with the largest delta (helps if neighbor crosstalk barely crosses).
3. Once locked: `showOnlyLed(i, CRGB::Green); FastLED.show();`. Print `"key " + i + " -> mux=" + (electrical/16) + " ch=" + (electrical%16) + " (electrical=" + electrical + ")"`.
4. Wait for release: loop until that channel's reading is within `kReleaseDelta` of `baseline[electrical]` for `kReleaseMs` continuously. (Optional but prevents the next-key wait from being polluted by a still-held key.)

**emitArray():**
Print verbatim:
```
inline constexpr uint8_t k3dot0Keys[32] = {
    v0, v1, v2, v3, v4, v5, v6, v7,
    v8, v9, v10, v11, v12, v13, v14, v15,
    v16, v17, v18, v19, v20, v21, v22, v23,
    v24, v25, v26, v27, v28, v29, v30, v31,
};
```
where `vN = discovered[N]`. Format with `printf`-style or `Serial.print` building each row. Print a blank line before and after the block so it's easy to copy.

**Naming:** snake_case for locals, camelCase or PascalCase for free functions (match T16 firmware convention seen in `src/main.cpp`).

**MODE handling note:** internal pull-up means GPIO0 reads HIGH when not pressed. The boot button on unwn_s3 is the same one the bootloader uses for download mode — holding it during normal boot just triggers our `ESP.restart()`. That's fine; on second boot the user releases it.

**Do NOT** add any MIDI, BLE, USB, filesystem, or configuration code. This is a single-purpose tool.
