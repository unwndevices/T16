<objective>
Fix CalibrationService to support multi-mux hardware (T32, 2 muxes, 32 keys). Currently `Adc::CalibrateMin/Max` and the no-arg `Adc::GetRaw()` hard-code `analogRead(_config._pin)`, which mirrors mux 0's commonPin. On T32, keys 16-31 belong to mux 1 (commonPin = GPIO17), so calibration and the hardware self-test sample the wrong physical signal for the upper half of the keyboard. Phase 12 already made `Adc::ReadValues()` multi-mux; this quick task extends the same logic to the calibration path.

Purpose: Production scan (`ReadValues`) is multi-mux but calibration is not, so on a freshly-flashed T32 the calibration min/max for keys 16-31 capture mux 0's idle/pressed values, then `_channels[16..31].minVal/maxVal` get loaded with wrong data — keys 16-31 will misbehave even after a correct calibration ritual.

Output: Multi-mux-aware Adc API + CalibrationService that uses it; `Key::mux_id` field populated at Adc init time; T16 build path unchanged at the byte level.

Note on `Key::mux_idx` semantics: After reviewing the code, `mux_idx` is the **logical key index** (used as `_channels[]` index via `adc->GetMux(0, mux_idx)`), NOT a 0..15 mux channel as the name suggests. The existing `SetMuxChannel(keys[i].mux_idx)` call in CalibrationService is therefore broken on T32 (passes values 16-31 to a function that masks to 4 bits). We will:
1. Add `Key::mux_id` (which mux owns this key, 0-indexed).
2. Reuse `mux_idx` as the channel-within-mux (0..15) — populated from the index at which `keyMapping[ch] == logical_key` for the owning mux. On T16 (identity mapping), `mux_idx` collapses to the logical index and behavior is unchanged.
</objective>

   ```cpp
   uint16_t Adc::CalibrateMin(uint8_t logical_key, uint8_t mux_id)
   {
       // NOTE: caller must already have called SetMuxChannel(channel_within_mux).
       uint8_t com_pin = (_mux_count > 0 && mux_id < _mux_count)
           ? static_cast<uint8_t>(_muxes[mux_id].commonPin)
           : _config._pin;
       uint i_v = 0;
       for (uint8_t j = 0; j < 16; j++)
       {
           i_v += analogRead(com_pin);
           delay(5);
       }
       i_v /= 16;
       _channels[logical_key].minVal = constrain(i_v, 0, 4095);
       return _channels[logical_key].minVal;
   }

   uint16_t Adc::CalibrateMax(uint8_t logical_key, uint8_t mux_id)
   {
       uint8_t com_pin = (_mux_count > 0 && mux_id < _mux_count)
           ? static_cast<uint8_t>(_muxes[mux_id].commonPin)
           : _config._pin;
       uint i_v = 0;
       for (uint8_t j = 0; j < 16; j++)
       {
           i_v += analogRead(com_pin);
           delay(5);
       }
       i_v /= 16;
       _channels[logical_key].maxVal = max((uint16_t)constrain(i_v, 0, 4095), _channels[logical_key].maxVal);
       return _channels[logical_key].maxVal;
   }

   uint16_t Adc::GetRawForMux(uint8_t mux_id) const
   {
       if (_mux_count > 0 && mux_id < _mux_count)
       {
           return analogRead(static_cast<uint8_t>(_muxes[mux_id].commonPin));
       }
       return analogRead(_config._pin);
   }
   ```
