    ```
    const BLE_PACKET_MAX = 20
    ```

    `getMidiTimestamp()`: returns [header, tsLow] from Date.now() & 0x1FFF. Header = ((ts >> 7) & 0x3F) | 0x80. TsLow = (ts & 0x7F) | 0x80.

    `frameSysExForBLE(manufacturerId: number, data: number[]): Uint8Array[]`:
    - Build full message: [0xF0, manufacturerId, ...data, 0xF7]
    - For short messages that fit in one packet (fullMessage.length <= BLE_PACKET_MAX - 3): produce single packet [header, ts, F0, ...middle_data, ts, F7] — note the timestamp byte before F7 per BLE MIDI spec
    - For long messages: first packet [header, ts, F0, data...] up to 20 bytes; continuation packets [header, data...] (no timestamp before data, byte index 1 has bit 7 clear); last packet must have [ts, F7] at end (timestamp before F7)
    - Every packet must be <= BLE_PACKET_MAX bytes

    `class BLESysExReassembler`:
    - Private buffer: number[], collecting: boolean
    - `feed(messages: Uint8Array[]): Uint8Array[]` — for each message: if starts with 0xF0, start buffer; if collecting, append; if ends with 0xF7, emit complete and reset
    - `reset(): void` — clear buffer and collecting flag

    `class BLEMidiTransport implements MidiTransport`:
    - Constructor takes BluetoothRemoteGATTCharacteristic
    - Private reassembler: BLESysExReassembler
    - Private listeners: Set of handler functions
    - `sendSysex(manufacturerId, data)`: frame with frameSysExForBLE, then for each packet call `characteristic.writeValueWithoutResponse(packet)` — MUST await each write sequentially to preserve ordering (use an internal async queue or fire-and-forget with ordering note)
    - Constructor sets up characteristic 'characteristicvaluechanged' event listener: on notification, call parseBLEMidiPacket(event.target.value), feed results to reassembler, emit complete SysEx messages to all listeners
    - `addSysexListener` / `removeSysexListener`: manage listeners set
    - `dispose()`: remove characteristic event listener, clear listeners
