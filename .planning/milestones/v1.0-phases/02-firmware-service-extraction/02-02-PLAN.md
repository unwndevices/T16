   ```cpp
   #pragma once
   #include <Arduino.h>

   namespace t16 {

   class MidiTransport {
   public:
       virtual ~MidiTransport() = default;
       virtual void sendNoteOn(uint8_t note, uint8_t velocity, uint8_t channel) = 0;
       virtual void sendNoteOff(uint8_t note, uint8_t velocity, uint8_t channel) = 0;
       virtual void sendControlChange(uint8_t cc, uint8_t value, uint8_t channel) = 0;
       virtual void sendPitchBend(int bend, uint8_t channel) = 0;
       virtual void sendAfterTouch(uint8_t note, uint8_t pressure, uint8_t channel) = 0;
       virtual void sendSysEx(size_t size, const byte* data) = 0;
       virtual void read() = 0;
       virtual void setHandleSystemExclusive(void (*function)(byte*, unsigned)) = 0;
   };

   } // namespace t16
   ```

2. Inside `src/Libs/MidiProvider.cpp`, create three concrete transport classes (private to the .cpp, not in the header):
   - `UsbMidiTransport` wrapping the USB MIDI interface
   - `BleMidiTransport` wrapping the BLE MIDI interface
   - `SerialMidiTransport` wrapping the TRS/Serial MIDI interface
   Each implements the MidiTransport virtual methods by delegating to the underlying library objects.

3. Refactor `MidiProvider` class:
   - Keep the same PUBLIC API (SendNoteOn, SendNoteOff, etc.) so callers don't change
   - Replace the three bool flags (midiBle, midiThru, midiOut) with:
     ```cpp
     std::array<MidiTransport*, 3> transports_;
     uint8_t activeCount_ = 0;
     ```
   - `Init()`: create all three transport objects, add USB transport to active list by default
   - `SetMidiBle(bool)`, `SetMidiOut(bool)`, `SetMidiThru(bool)`: rebuild the active transports list based on enabled flags
   - Each `Send*` method loops over `transports_[0..activeCount_-1]` calling the virtual method
   - `Read()`: loops over all transports calling `read()`
   - Keep note_pool, chord_pool, strum_pool in MidiProvider (note tracking is transport-agnostic)
   - Keep the concrete MIDI library objects (usb_midi, MIDI_USB, MIDI_BLE, MIDI_SERIAL etc.) as private members of MidiProvider — the transport wrappers reference them
   - The transport wrapper classes can be nested private classes or defined in the .cpp file

4. Keep `SetHandleSystemExclusive` working — it needs to register the callback on the USB transport (SysEx only comes via USB per current behavior).

5. Ensure MidiProvider's public API signature does NOT change. Callers (main.cpp, SysExHandler) must compile without modification.
