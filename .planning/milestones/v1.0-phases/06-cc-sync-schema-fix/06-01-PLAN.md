Add missing `pal` (palette) field to the JSON schema, regenerate TypeScript types, update defaults and migration so that device-originated configs validate and import correctly.

Purpose: Firmware serializes `pal` per bank, but the schema has `additionalProperties: false` without listing it. This causes ajv to reject device configs on import, breaking config export-reimport.

Output: Schema, types, defaults, migration, and tests all include `pal`.
</objective>
