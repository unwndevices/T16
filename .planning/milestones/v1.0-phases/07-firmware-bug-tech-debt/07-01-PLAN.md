  <action>
Fix the UpdateTransition method in LedManager.cpp (line 259-266). The current code unconditionally overwrites currentPattern_ after the conditional move from nextPattern_. Add an `else` clause so the WaveTransition is only created when the condition is NOT met.

Current (broken):
```cpp
void LedManager::UpdateTransition()
{
    if (nextPattern_ && !nextPattern_->isTransition)
    {
        currentPattern_ = std::move(nextPattern_);
    }
    currentPattern_ = std::make_unique<WaveTransition>(Direction::DOWN);
}
```

Fixed:
```cpp
void LedManager::UpdateTransition()
{
    if (nextPattern_ && !nextPattern_->isTransition)
    {
        currentPattern_ = std::move(nextPattern_);
    }
    else
    {
        currentPattern_ = std::make_unique<WaveTransition>(Direction::DOWN);
    }
}
```

This preserves the existing behavior for callers that invoke UpdateTransition without a nextPattern_ (bank-switch visual transitions) while correctly completing transitions when nextPattern_ is ready.
