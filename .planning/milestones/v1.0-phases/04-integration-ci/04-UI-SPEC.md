# Phase 4 — UI Design Contract

### New Interaction: Upload Page Auto-Bootloader Flow

State machine for the updated Upload page:

| State | Button Label | Button Variant | Progress | Instructions Visible |
|-------|-------------|----------------|----------|---------------------|
| idle (no MIDI) | "Connect Device" | secondary | hidden | Manual fallback only |
| idle (MIDI connected) | "Update Firmware" | primary | hidden | Auto-bootloader steps |
| entering bootloader | "Entering Update Mode..." | primary, disabled | hidden | Auto-bootloader steps |
| bootloader ready | "Upload Firmware" | primary | hidden | Auto-bootloader steps |
| uploading | "Uploading..." | primary, disabled | visible (progress bar) | Auto-bootloader steps |
| success | "Update Firmware" | primary | hidden, replaced by success toast | Auto-bootloader steps |
| error | "Update Firmware" | primary | hidden, replaced by error toast | Auto-bootloader steps + fallback link |

### New Interaction: Config Import Validation

Error display uses existing Card component with error styling:

```
+--------------------------------------------------+
| Card: Invalid Configuration File                  |
|                                                   |
|  The file could not be imported. See details:     |
|                                                   |
|  - banks[0].scale: must be 0-11                  |
|  - global.midi_channel: missing required field    |
|  - global.velocity_curve: must be 0-4            |
|                                                   |
+--------------------------------------------------+
```

- Error card background: `color-mix(in srgb, var(--color-error) 8%, transparent)`
- Error card border: `1px solid color-mix(in srgb, var(--color-error) 30%, transparent)`
- Error card border-radius: `var(--radius-md)` (8px)
- Error list font: `--font-mono` at `--text-xs` (12px), `--color-error` color
- Error list item spacing: `var(--space-xs)` (4px) between items
