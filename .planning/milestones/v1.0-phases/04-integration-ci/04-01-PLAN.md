<task type="auto">
  <name>Task 2: Create CI workflow (.github/workflows/ci.yml)</name>
  <files>.github/workflows/ci.yml</files>
  <read_first>
    - .github/workflows/deploy.yml (existing workflow for reference)

  </read_first>
  <action>
    Create `.github/workflows/ci.yml` with three jobs:

    **Trigger:** `on: push: branches: ['**']` and `on: pull_request: branches: [main]`

    **Job 1: firmware-build**
    - runs-on: ubuntu-latest
    - steps:
      - `actions/checkout@v4`
      - `actions/cache@v4` with path `~/.platformio`, key `pio-${{ hashFiles('platformio.ini') }}`, restore-keys `pio-`
      - `actions/setup-python@v5` with python-version `'3.11'`
      - `pip install platformio`
      - `pio run -e esp32s3` (build firmware)
      - `pio test -e native` (run unit tests)

    **Job 2: firmware-format**
    - runs-on: ubuntu-latest
    - steps:
      - `actions/checkout@v4`
      - Install clang-format: `sudo apt-get install -y clang-format`
      - Check formatting: `find src/ -name '*.hpp' -o -name '*.cpp' -o -name '*.h' | xargs clang-format --dry-run --Werror`

    **Job 3: web-build**
    - runs-on: ubuntu-latest
    - defaults: run: working-directory: editor-tx
    - steps:
      - `actions/checkout@v4`
      - `actions/setup-node@v4` with node-version `'22'`, cache `'npm'`, cache-dependency-path `editor-tx/package-lock.json`
      - `npm ci`
      - `npx tsc --noEmit` (typecheck)
      - `npm run lint` (eslint)
      - `npx prettier --check .` (formatting)
      - `npm run test -- --run` (vitest, non-watch mode)
      - `npm run build` (vite build)

    All three jobs are independent (no `needs:` between them) so they run in parallel.

    Do NOT modify deploy.yml -- keep it as-is for FTP deploy on main.
  </action>
