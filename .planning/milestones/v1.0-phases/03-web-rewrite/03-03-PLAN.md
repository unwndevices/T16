1. Create `editor-tx/src/contexts/ConnectionContext.tsx` (per D-05):

Manages WebMIDI lifecycle. Provides:
- `input: Input | null`, `output: Output | null`, `isConnected: boolean`, `isDemo: boolean`
- `connect(): Promise<void>` -- calls `enableMidi()` from services, then `findDevice()`, sets state. Must only be called from user gesture (button click).
- `disconnect(): void` -- calls `disableMidi()`, clears state.
- `setDemo(demo: boolean): void`
- Listens for WebMIDI `connected`/`disconnected` events to update state.
- Uses React 19 context pattern: `<ConnectionContext value={state}>` (not .Provider).
- Auto-reconnect on device reconnection event.

2. Create `editor-tx/src/contexts/ConfigContext.tsx` (per D-05):

Manages device configuration. Provides:
- `config: T16Configuration` -- editor-side desired state
- `deviceConfig: T16Configuration | null` -- confirmed device state
- `selectedBank: number`
- `isSynced: boolean` -- computed from deep comparison of config vs deviceConfig
- `setBank(bank: number): void`
- `updateParam(domain: number, bank: number, field: number, value: number): void` -- updates local config AND sends SysEx via services/midi.ts using the output from useConnection()

Use `useReducer` with `ConfigAction` type from types/midi.ts. The reducer handles SET_CONFIG, SET_DEVICE_CONFIG, UPDATE_PARAM (updates specific field in config), SET_BANK, SYNC_CONFIRMED.

ConfigContext must be nested inside ConnectionContext (needs `useConnection()` for output reference).

On connect: register SysEx listener on input to receive config dumps and version responses. Parse with `parseConfigDump()` from services/midi.ts. Dispatch SET_DEVICE_CONFIG and SET_CONFIG.

On disconnect: clear deviceConfig to null, set isSynced false.

Create a default config matching the structure in types/config.ts (copy the default values from current MidiProvider.jsx).

3. Create `editor-tx/src/contexts/ToastContext.tsx`:

Wraps Radix Toast (from `radix-ui`) to provide imperative toast API similar to current Chakra useToast. Provides:
- `toast(options: { title: string; description?: string; variant?: 'default' | 'error' | 'success' }): void`
- Manages a toast queue (array of active toasts)
- Renders `<Toast.Provider>` and `<Toast.Viewport>` from Radix Toast
- Auto-dismiss after 5 seconds
