1. Create `src/SysExHandler.hpp` with `#pragma once`:

   ```cpp
   #pragma once
   #include <Arduino.h>

   class ConfigManager;
   class MidiProvider;

   class SysExHandler {
   public:
       SysExHandler(ConfigManager& configManager, MidiProvider& midiProvider);

       // Main entry point — called from MidiProvider's SysEx callback
       void ProcessSysEx(byte* data, unsigned length);

   private:
       ConfigManager& configManager_;
       MidiProvider& midiProvider_;

       // Firmware version — set to cfg.version at construction or via setter
       uint8_t firmwareVersion_ = 0;

       void HandleVersionRequest();
       void HandleConfigDumpRequest();
       void HandleConfigLoad(const byte* payload, size_t payloadLen);
       void HandleParamSet(const byte* payload, size_t payloadLen);
       void HandleCalibrationReset();
       void SendAck(uint8_t cmd, uint8_t status);
   };
   ```

2. Create `src/SysExHandler.cpp`:

   ```cpp
   #include "SysExHandler.hpp"
   #include "SysExProtocol.hpp"
   #include "ConfigManager.hpp"
   #include "Libs/MidiProvider.hpp"
   ```

   **Constructor:** Store references to configManager and midiProvider. Set `firmwareVersion_ = configManager.Global().version`.

   **ProcessSysEx(data, length):**
   Per PROTO-04 — validate before processing:
   1. Check `length < SysEx::MIN_MESSAGE_LENGTH` — if so, `log_d("SysEx too short: %u bytes", length)` and return
   2. Check manufacturer ID: `data[1] != SysEx::MANUFACTURER_ID` — if so, return silently (message is not for us)
   3. Extract: `uint8_t cmd = data[2]; uint8_t sub = data[3];`
   4. Payload starts at `data + 4`, length is `length - 4`

   NOTE on byte layout: The arduino_midi_library handler receives data where data[0] = 0xF0, data[1] = first data byte. Since our messages are F0 7D cmd sub [payload] F7, data[1] = 0x7D (manufacturer ID), data[2] = cmd, data[3] = sub. The research has an open question about exact offsets — add a debug log `log_d("SysEx: len=%u data[0]=0x%02X data[1]=0x%02X data[2]=0x%02X data[3]=0x%02X", length, data[0], data[1], data[2], data[3])` at the start of ProcessSysEx for runtime verification. This log can be removed once offsets are confirmed.

   5. Switch on `cmd`:
      - `SysEx::CMD_VERSION`: if sub == SUB_REQUEST, call `HandleVersionRequest()`
      - `SysEx::CMD_CONFIG`: if sub == SUB_REQUEST, call `HandleConfigDumpRequest()`. If sub == SUB_LOAD, call `HandleConfigLoad(payload, payloadLen)`
      - `SysEx::CMD_PARAM`: if sub == SUB_REQUEST and payloadLen >= 3, call `HandleParamSet(payload, payloadLen)`
      - `SysEx::CMD_CALIBRATION`: if sub == SUB_REQUEST, call `HandleCalibrationReset()`
      - default: `log_d("Unknown SysEx command: 0x%02X", cmd)`

   **HandleVersionRequest():** Per PROTO-05
   - Build response: `byte response[] = { SysEx::MANUFACTURER_ID, SysEx::CMD_VERSION, SysEx::SUB_RESPONSE, SysEx::PROTOCOL_VERSION, firmwareVersion_ }`
   - Call `midiProvider_.SendSysEx(sizeof(response), response)`
   - `log_d("Version response sent: proto=%d fw=%d", SysEx::PROTOCOL_VERSION, firmwareVersion_)`

   **HandleConfigDumpRequest():** Per PROTO-03
   - Allocate buffer: `char buffer[2048]`
   - Write header bytes: `buffer[0] = SysEx::MANUFACTURER_ID; buffer[1] = SysEx::CMD_CONFIG; buffer[2] = SysEx::SUB_RESPONSE;`
   - Serialize config: `size_t jsonLen = configManager_.SerializeToBuffer(buffer + 3, sizeof(buffer) - 3)`
   - Send: `midiProvider_.SendSysEx(jsonLen + 3, reinterpret_cast<byte*>(buffer))`
   - `log_d("Config dump sent: %u bytes", jsonLen)`

   **HandleConfigLoad(payload, payloadLen):** Per PROTO-03
   - Validate payloadLen > 0, else log error and SendAck(CMD_CONFIG, STATUS_ERROR) and return
   - Call `bool ok = configManager_.DeserializeFromBuffer(reinterpret_cast<const char*>(payload), payloadLen)`
   - If ok: `SendAck(SysEx::CMD_CONFIG, SysEx::STATUS_OK)` and `log_d("Config loaded from editor")`
   - If !ok: `SendAck(SysEx::CMD_CONFIG, SysEx::STATUS_ERROR)` and `log_d("Config load failed: parse error")`

   **HandleParamSet(payload, payloadLen):** Per PROTO-02
   - Extract: `uint8_t domain = payload[0]; uint8_t bankIndex = payload[1]; uint8_t fieldId = payload[2]; uint8_t value = payload[3]` (need payloadLen >= 4 for single-byte values)
   - If payloadLen < 4: log error and return
   - Switch on domain:
     - `SysEx::DOMAIN_GLOBAL`: call `configManager_.SetGlobalParam(fieldId, value)`
     - `SysEx::DOMAIN_BANK_KB`: validate bankIndex < BANK_AMT, call `configManager_.SetBankParam(bankIndex, fieldId, value)`
     - `SysEx::DOMAIN_BANK_CC`: validate bankIndex < BANK_AMT, need payloadLen >= 5 for channel+id. `uint8_t ccIndex = fieldId; uint8_t channel = value; uint8_t id = payload[4]`. Call `configManager_.SetCCParam(bankIndex, ccIndex, channel, id)`
     - default: log unknown domain
   - Send ACK: `SendAck(SysEx::CMD_PARAM, SysEx::STATUS_OK)`
   - `log_d("Param set: domain=%d bank=%d field=%d value=%d", domain, bankIndex, fieldId, value)`

   **HandleCalibrationReset():**
   - Log: `log_d("Calibration reset requested")`
   - This will eventually call calibration.Delete() + ESP.restart(), but for now just log — the actual calibration manager is a separate concern (D-07). Add a TODO comment: `// TODO: Wire to CalibrationManager when separated (D-07, Phase 2)`

   **SendAck(cmd, status):**
   - `byte ack[] = { SysEx::MANUFACTURER_ID, cmd, SysEx::SUB_ACK, status }`
   - `midiProvider_.SendSysEx(sizeof(ack), ack)`
